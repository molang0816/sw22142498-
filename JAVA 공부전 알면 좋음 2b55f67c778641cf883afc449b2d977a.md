# JAVA 공부전 알면 좋음

- **컴파일** : 인간 언어를 기계어로 번역하는 과정
- **컴파일러** : 컴파일 하는 프로그램
- **디버깅 :** 버그(시스템의 논리적인 오류나 비정상적 연산)를 고치는 과정
- **바이트** / **바이너리 코드** : 0과 1로 이루어진 코드 (기계어)
- **기계어** : CPU가 명령 처리시 사용하는 언어 (2진법 구성)
- **어셈블리어** : 사람이 이해하기 쉽게 기계어에서 단어로 만든것 ex: mov eax , add eax… (저급 언어)
- **고급 언어** (High-Level language) : 사람이 이해하기 편한 프로그래밍 언어 (C, C++, C#, Java)

---

- **컴파일 언어** (Compiled Language) 
**:** 소스코드를 한꺼번에 다른 목적 코드로 번역한 후, 한 번에 실행하는 언어
**-** CPU가 알아들을 수 있는 로우 레벨 언어 (어셈블리어, object 코드, machine code) 로 번역됨 
  이후 컴파일하면 실행 가능한 (바이너리) 파일이 생성됨 → 번역과 실행이 따로 이루어짐
**-** 코드를 다 적고 나서 한꺼번에 검사하는 형태
**-** 규모가 큰 소스의 경우, 컴파일된 프로그램의 경우 일반적으로 인터프리터를 이용해 실행시키
  는 것보다 훨씬 빠르게 동작 하지만 컴파일 과정에서는 시간이 많이 소요, 메모리 많이 차지
**-** 소스코드를 기계어로 번역하는 과정이 빌드 과정, 고급언어 ---(변환)--> 로우 레벨 언어 이러한
  빌드 과정을 거쳐 실행파일 생성
- ex : C/ C++, C#, Objective C
- **인터프리터 언어** (Interpreter Language)
****: ****소스코드를 한 줄 한 줄 읽어가며 명령을 바로 처리하는 언어
**-** 인터프리터(interpreter)란, 우리말로 '통역사'라는 뜻임. 우리가 컴퓨터에 명령을 내리면 한 줄
  씩  즉시 통역사가 컴퓨터 언어로 바꿔주는 거라고 생각하면 됨 **(컴파일 필요 없음**)
**-** 규모가 큰 소스 경우, 컴파일 언어는 프로그램 실행 전 먼저 기계어로 컴파일하는 데 시간이
  오래 걸릴 수 있지만 인터프리터 언어는 코드를 한 번에 한 줄씩 읽어 바로 실행이 가능
**-** 한 줄 씩 명령을 내리다 보니, 그 명령 자체의 속도는 컴파일러 언어에 비해 느림
**-** 인터프리터 언어는 고급 프로그램을 즉시 실행시킬 수 있어 프로그램 수정이 간단함
**-** ex : 자바스크립트, 파이썬, 루비, sql
- Java는 컴파일, 인터프리터 둘다 사용

---

- **C언어** 경우 각각의 운영체제 마다 컴파일 결과물 다름 → 컴파일러 여러개 있어야함
- 하지만 **Java**의 경우 운영체제마다 각각의 **JVM**(운영체제 사이에서 호환, 번역 시켜주는 역할)이 있어 컴파일러 하나만 있어도 컴파일 결과물 같음 (Java 설치시 같이 깔림)
- 그루비, 스칼라, 코틀린도 JVM 사용함
- **LTS** (Long Time Support) : 오래 지원되는 버전
- **환경변수** : 프로세스가 컴퓨터에서 동작하는 방식에 영향을 미치는, 동적인 값들의 모임
- **Path :** 디렉터리 경로의 목록. 사용자가 전체 경로를 지정하지 않고 명령을 입력하면 이 목록을
          확인하여 해당 명령어가 경로에 속하는지를 살펴봄

- **JDK** (Java Develoment Kit)
JDK = JRE + 개발을 위한 도구
→ 컴파일러, 디버그 도구 등
→ JAVA버전 = JDK 버전
- **JRE** (Java Runtime Environmeant)
JRE = JVM + 자바 프로그램 실행에 필요한
라이브러리 파일 등
JVM의 실행 환경 구현 (여러 라이브러리)
- **JVM** (Java Virtual Machine)
: 자바 가상 머신으로 OS별로 존재, 
  바이너리 코드 읽고, 검증, 실행함

- **JDK** > **JRE** > **JVM** 
(JDK설치시 하위도 같이 설치됨)

- **Oracle JDK :** 오라클에서 만든 JDK / 개인 무료, 기업 유료
- **Open** **JDK** : Oracle JDK와 비슷한 성능 / 언제나 무료
- **-** JDK는 각 버전마다 새로운 기능 추가 되거나 사라짐
**-** JDK는 종류가 있고, 기능은 동일하나 성능 비용 차이가 약간 있음
- **빌드** (Build) : 소스 코드 파일을 컴퓨터에서 실행할 수 있게 독립 SW 가공물로 변환시키는 과정
** 독립 SW 가공물 = Artifact
** 빌드 과정 : 소스 코드 컴파일 → 테스트 코드 컴파일 → 테스트 코드 실행 → 테스트 코드
                      리포트 작성 → 추가 설정 작업 진행 → 패키징 수행 → 최종 Artifact 만들어 냄
** 테스트 코드 : 소프트웨어의 기능과 동작을 테스트하는 데 사용되는 코드
                           - 테스트 코드는 개발자가 작성한 코드를 실행하고 예상된 결과가 나오는지
                             확인하는 데 사용됨
                           - 디버깅 절약, 테스트 자동화
                           - 테스트 코드 유/무에 따라 개발 속도, 안정성, 품질 차이 남
- **빌드 툴** (Build Tool)
- 소스 코드의 빌드 과정 자동 처리 해주는 프로그램
- 외부 소스 코드 (외부 라이브러리) 자동 추가, 관리
- ANT, Maven, **Gradle** 이 있음 (Gradle 빌드 툴을 많이 사용)
